# POST SIGNATURE IMMUTABILITY

Version: v1.0  
Estado: CANONICO  
Normas: MUST, SHOULD, MAY

## 0. Proposito
Dejar explicita la inmutabilidad posterior a la firma para evitar confusiones.

## 1. Regla principal
- MUST: Un documento firmado no puede modificarse.
- MUST: Cualquier cambio requiere un nuevo documento y un nuevo workflow.

## 2. UI obligatoria
- MUST: En detalle mostrar el texto:
  "Una vez firmado, este documento no puede modificarse.
   Cualquier cambio requiere un nuevo flujo."

## 3. No-responsabilidades
- No define editor ni versiones.
